## 二、发现并修复的页面 Bug

### Bug #1: 功率一览表被 `resultArea` 遮蔽

| 项目 | 内容 |
|------|------|
| **严重程度** | 🔴 功能不可用 |
| **描述** | `catalogToggle` 和 `catalogBody`（功率一览表）嵌套在 `#resultArea` 内部，而 `resultArea` 默认 `display:none`。用户在未执行任何计算前，无法看到也无法操作功率一览表。 |
| **根因** | HTML 结构问题：`resultArea` 的 `</div>` 闭合标签位置不对，把独立的功率一览表也包进去了。 |
| **修复** | 将 `resultArea` 的 `</div>` 上移至 catalog 区域之前，使 `catalogToggle` 和 `catalogBody` 脱离 `resultArea`，始终可见。 |

### Bug #2: 输入数字反序问题（上次会话修复）

| 项目 | 内容 |
|------|------|
| **严重程度** | 🟠 影响用户体验 |
| **描述** | 用户鼠标全选输入框中的已有数值后直接键入新值，数字出现反序。 |
| **根因** | 浏览器 `<input type="number">` 的光标/选区处理 bug。 |
| **修复** | 将所有数值输入框改为 `type="text"` + `inputmode="decimal"`，用 JS 正则过滤非数字字符。 |
